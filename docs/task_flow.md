# Task Flow

## Developer Workflow
1.  **Launch LogKitty:** User opens the app.
2.  **Grant Permissions:** User grants Overlay and Accessibility permissions (and Read Logs via ADB if needed).
3.  **Start Service:** The overlay appears.
4.  **Debug Target App:** User switches to the app they want to debug.
5.  **View Logs:** User expands the LogKitty sheet to see logs in real-time.
6.  **Filter:** User types in the box to filter logs (Future).
7.  **Copy:** User taps copy to save a stack trace.
